Umfeld des Projekts

Ist-Zustand

    Situation:

  • Wachsende Server Infrastruktur
  • Wachsende Anzahl der Systembetreuer
  • Komplexere Kundensetups
  • Sensible Dienste und Applikationen
  • Mühsames Wiederherstellen alter Konfigurationen
  • Schwere Nachvollziehbarkeit innerhalb der Systeme
  • Keine einheitliche Sammelstelle für Konfigurationen vorhanden

Soll-Zustand

    Anforderungen

  • Generelles Konzept zur Serveradministration
  • Nachvollziehbarkeit
  • Wiederherstellbarkeit
  • Sicherheit
  • Zentrale Sammelstelle für Konfigurationen
<

Was ist ein Versionskontrollsystem?

    Versionskontrollsysteme (oder Versionsverwaltungssysteme):

  • Generell für Betreuung von Software Projekten entwickelt
  • Ermöglicht verteiltes Arbeiten mehrerer Entwickler
  • Bietet Werkzeuge für professionelle Software Entwicklung

    Klassische Funktionen eines VCS:

  • Splitten separater Arbeitsschritte via "Commits"
  • History bzw. Verlauf des Projekts
  • Änderungen zwischen Commits anzeigen
  • Austausch von getätigten Commits zu anderen Entwicklern
